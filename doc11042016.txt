参考issue： MEDI-12039

issue背景：Enhancement for the SEP question section (2017)，修改了结点的XPath。页面选项从多选变成单选。

我的business逻辑理解：
页面填写信息 -> 信息存在xml文件中 -> transfer方法取xml中的信息 -> 成功执行 -> 将结果存在oec文件中 -> 将oec文件发送至FTP

针对issue：
	* SEP有两种类型，一种带日期，一种不带日期，看代码发现他们都是存在同一个xml结点中，因此可以将此分为两个等价类，只需要走两个case，节约时间。
	* 在页面上选了SEP后，点击next（触发代码），再查看xml检查是否是想要的结果。
	* 最理想的做法，找到xml文件的位置，直接修改xml文件中SEP节点的值，自己写一个方法调用transfer的代码，打印查看mapping结果是否正确。

5.给sublime小白的快捷键：
	* 用 Ctrl + P 可以快速跳转到当前项目中的任意文件，可进行关键词匹配。
	* 用 Ctrl + P 后 @ (或是Ctrl +R)可以快速列出/跳转到某个函数
	* 用 Ctrl + P 后 # 可以在当前文件中进行搜索。
	* 用 Ctrl + P 后 : (或是Ctrl+G)加上数字可以跳转到相应的行。
	* 用 Ctrl +P 加上一些关键词跳转到某个文件同时加上 @ 来列出/跳转到目标文件中的某个函数，或是同时加上 # 来在目标文件中进行搜索，或是同时加上 : 和数字来跳转到目标文件中相应的行。
	* 按住Ctrl，然后在页面中希望中现光标的位置点击，实现多重选择功能，允许在页面中同时存在多个光标。
